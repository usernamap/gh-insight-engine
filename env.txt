# =============================================================================
# NOTES IMPORTANTES
# =============================================================================
# 
# 1. GITHUB TOKEN :
#    - Créez un token GitHub Classic sur : https://github.com/settings/tokens
#    - Permissions requises : repo, user, read:org
#    - Expiration recommandée : 90 jours
#
# 2. OPENAI API KEY :
#    - Créez une clé sur : https://platform.openai.com/api-keys
#    - Assurez-vous d'avoir des crédits suffisants
#
# 3. SCHEDULING :
#    - SCHEDULE_ENABLED=true active les mises à jour automatiques
#    - SCHEDULE_AUTH_TOKEN doit être un JWT valide obtenu via /auth/login
#    - Le scheduling utilise l'utilisateur défini dans GITHUB_USERNAME
#
# 4. SÉCURITÉ :
#    - JWT_SECRET doit être long et aléatoire
#    - Ne partagez jamais vos tokens GitHub ou OpenAI
#    - En production, utilisez des variables d'environnement sécurisées
#
# 5. BASE DE DONNÉES :
#    - MongoDB doit être installé et accessible
#    - L'URL doit inclure le nom de la base de données
#    - En production, utilisez MongoDB Atlas ou un service cloud
#
