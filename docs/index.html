GitHub Insight Engine API - Simplified (1.0.0)

Download OpenAPI specification:

URL: https://github.com/org/gh-insight-engine License: MIT

API REST simplifiée pour l'analyse en profondeur des données GitHub.

Cette API permet d'analyser les profils GitHub et repositories avec une approche tout-en-un.

Documentation complète de l'API

🔐 Authentification

L'API utilise l'authentification JWT Bearer Token.

Workflow d'authentification :

  1. Connexion initiale : Utilisez l'endpoint POST /auth/login avec votre token GitHub Classic
  2. Endpoints protégés : Utilisez le token JWT pour accéder aux données complètes

Configuration du token GitHub Classic :

Votre token doit avoir les permissions suivantes :

  • repo (accès aux repositories privés)
  • user (lecture du profil utilisateur)
  • read:org (lecture des organisations)

🚀 Endpoints Optimisés (Collecte vs Consultation)

Architecture en deux phases :

📥 Phase 1 : Collecte (POST)

  • POST /users/{username} - Collecte des données utilisateur depuis GitHub API
  • POST /repositories/{username} - Collecte des repositories avec analyses DevOps

📊 Phase 2 : Consultation (GET)

  • GET /users/{username} - Consultation des données utilisateur stockées
  • GET /repositories/{username} - Consultation des repositories stockés
  • POST /auth/login - Authentification

⚡ Avantages de cette approche

  • Performance : Consultation ultra-rapide depuis la base de données
  • Contrôle : Collecte à la demande pour des données fraîches
  • Flexibilité : Données publiques accessibles sans authentification
  • Efficacité : Évite les appels répétés à l'API GitHub

Rate Limiting

  • Global: 100 requêtes par 15 minutes par IP
  • Authentification: 5 tentatives par 15 minutes par IP

🔄 Workflow recommandé

  1. Authentification → POST /auth/login
  2. Collecte des données → POST /users/{username} puis POST /repositories/{username}
  3. Consultation rapide → GET /users/{username} et GET /repositories/{username}
  4. Mise à jour → Re-collecte avec POST si les données sont anciennes

Authentication

Authentification et gestion des sessions

Authentification avec token GitHub

Authentifie un utilisateur avec son token GitHub Classic et retourne un token JWT.

Permissions GitHub requises :

  • repo : Accès aux repositories privés
  • user : Lecture du profil utilisateur
  • read:org : Lecture des organisations

Workflow :

  1. Générez un token GitHub Classic sur https://github.com/settings/tokens
  2. Cochez les permissions requises
  3. Utilisez ce token dans le champ githubToken
  4. Récupérez le JWT dans la réponse pour les appels suivants
Request Body schema: application/json
required

Collecte des données GitHub d'un utilisateur

Déclenche la collecte et le stockage des données GitHub d'un utilisateur depuis l'API GitHub.

🔐 Authentification requise - L'utilisateur ne peut collecter que ses propres données.

  • Enrichissement avec données DevOps complètes

  • Stockage en base de données

Repositories

Gestion des repositories avec analyses DevOps complètes

Collecte des repositories d'un utilisateur

Déclenche la collecte et le stockage des repositories d'un utilisateur depuis l'API GitHub.

🔐 Authentification requise - L'utilisateur ne peut collecter que ses propres repositories.

Processus de collecte :

  • Récupération de tous les repositories (utilisateur + organisations)
  • Filtrage intelligent (contributeur actif)
  • Enrichissement avec données DevOps complètes
  • Calcul des métriques et analyses
  • Stockage en base de données

Données collectées :

  • ✅ Données DevOps (GitHub Actions, sécurité, packages, protection branches)
  • ✅ Métriques de performance et qualité
  • ✅ Statistiques de langages et topics
  • ✅ Scores de maturité DevOps
  • ✅ Santé communautaire
  • ✅ Données de trafic
