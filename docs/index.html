GitHub Insight Engine API (0.1.9)

Download OpenAPI specification:

URL: https://github.com/org/gh-insight-engine License: MIT

API REST pour l'analyse en profondeur des données GitHub avec insights IA.

Cette API permet d'analyser les profils GitHub, repositories et de générer des insights quantitatifs et qualitatifs grâce à l'intelligence artificielle.

Documentation complète de l'API

🔐 Authentification et Contexte Persistant

L'API utilise l'authentification JWT Bearer Token. Le contexte d'authentification est automatiquement persisté lors des tests de documentation pour une expérience utilisateur optimale.

Workflow d'authentification :

  1. Connexion initiale : Utilisez l'endpoint POST /auth/login avec votre token GitHub Classic
  2. Contexte automatique : Le token JWT retourné est automatiquement utilisé pour tous les endpoints suivants
  3. Persistance de session : Votre authentification reste active pendant toute la session de test
  4. Endpoints protégés : Testez directement les endpoints privés sans re-saisir le token

Configuration du token GitHub Classic :

Votre token doit avoir les permissions suivantes :

  • repo (accès aux repositories privés)
  • user (lecture du profil utilisateur)
  • read:org (lecture des organisations)

Test de la documentation :

  1. 🚀 Commencez par /auth/login avec vos identifiants
  2. 📋 Copiez le token depuis tokens.accessToken dans la réponse
  3. 🔐 Cliquez sur "Authorize" en haut de cette page
  4. 📝 Collez le token JWT (sans "Bearer ") et validez
  5. ✅ Tous les endpoints protégés utilisent maintenant votre token automatiquement
  6. 📊 Explorez l'API complète avec votre contexte utilisateur authentifié

Rate Limiting

  • Global: 100 requêtes par 15 minutes par IP
  • Authentification: 5 tentatives par 15 minutes par IP
  • Analyses: 10 analyses par heure par IP

Workflow d'analyse recommandé

  1. Authentification → POST /auth/login
  2. Vérification → GET /auth/validate
  3. Profil utilisateur → GET /users/{username}
  4. Analyse complète → POST /analytics/{username}/analyze
  5. Insights IA → POST /insights/{username}/generate
  6. Consultation des résultats → Endpoints GET correspondants
