📝 Guide : Création de votre Token GitHub Classic

Suivez ce guide étape par étape pour créer votre token d'authentification GitHub

1

Accéder aux paramètres GitHub

Connectez-vous à GitHub.com et cliquez sur votre photo de profil en haut à droite, puis sélectionnez "Settings".

GitHub.com → Photo de profil → Settings
2

Naviguer vers Developer Settings

Dans le menu de gauche, faites défiler jusqu'en bas et cliquez sur "Developer settings".

Settings → Developer settings (en bas du menu)
3

Accéder aux Personal Access Tokens

Cliquez sur "Personal access tokens" puis sélectionnez "Tokens (classic)".

Personal access tokens → Tokens (classic)
4

Générer un nouveau token

Cliquez sur "Generate new token" puis "Generate new token (classic)".

💡 Important : GitHub peut vous demander votre mot de passe ou une authentification 2FA à cette étape.
5

Configuration du token

Remplissez les informations suivantes :

Note : GitHub Insight Engine
Expiration : 90 days (recommandé)
6

Sélection des permissions (Scopes)

Cochez exactement ces permissions :

user - Accès complet aux informations du profil utilisateur
public_repo - Accès aux repositories publics
read:org - Lecture des organisations et équipes
user:email - Accès aux adresses email
repo - Accès aux repositories privés (optionnel)
⚠️ Attention : Ne sélectionnez pas d'autres permissions pour des raisons de sécurité.
7

Générer et copier le token

Cliquez sur "Generate token" en bas de la page. Copiez immédiatement le token généré.

🚨 IMPORTANT : Le token ne sera affiché qu'une seule fois ! Copiez-le immédiatement et conservez-le en sécurité.

🔒 Sécurité de votre token

  • Ne partagez jamais votre token avec qui que ce soit
  • Gardez votre token confidentiel et en lieu sûr
  • Si votre token est compromis, révoquez-le immédiatement sur GitHub
  • Votre token est utilisé uniquement pour l'analyse de votre profil
  • Nous ne stockons jamais votre token de façon persistante

🛠️ Résolution de problèmes

❓ Le token ne fonctionne pas

Vérifiez que vous avez sélectionné les bonnes permissions et que le token n'a pas expiré.

❓ Je ne trouve pas "Developer settings"

Assurez-vous d'être connecté à GitHub.com et regardez tout en bas du menu de gauche dans Settings.

❓ Erreur "Token invalid"

Le token est peut-être incorrect ou a expiré. Générez un nouveau token en suivant les étapes ci-dessus.
